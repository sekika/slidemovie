# AI による原稿生成

Markdown の原稿を手動で書くのは時間がかかることがあります。**ChatGPT**、**Claude**、**Google Gemini** などの大規模言語モデル (LLM) を使って、`slidemovie` 用の Markdown を自動生成させることができます。

## プロンプト

以下のプロンプトをコピーして、AI のチャット画面に貼り付けてください。一番下の `(ここに動画化したいドキュメントやテキストを貼り付けてください)` の部分を、実際に動画にしたいテキストに置き換えてください。

### プロンプト・テンプレート

```text
Pythonプログラム「slidemovie」を紹介する動画を作成します。このプログラムの目的と、何も知らない人にとってどのように役立つかを説明することが目標です。技術的な詳細はマニュアルを参照できるため、省略しても構いません。

目的は、以下のドキュメントに基づいて、スライド作成とAIナレーション用の「読み上げ原稿」を含むMarkdownファイルを生成することです。

以下の「Markdown作成ルール」と「ドキュメント」に基づいて、**Pandocを使ってPowerPoint (pptx) に変換可能なMarkdownファイル**を作成してください。

## Markdown作成ルール

1.  **読み上げ原稿（音声用）は `notes` ブロックの中だけに書いてください。**

    *   **`notes` ブロックは「完成された読み上げ原稿」にしてください**: 記号や注釈を含めず、ナレーターがそのまま読み上げられる形式（話し言葉）にしてください。
    *   スライド本文には「話し言葉」を書かないでください。

2.  **1スライド = 1トピック単位**

    *   原稿のトピックが変わるタイミングでスライドを分割してください。

3.  **標準的な Pandoc Markdown 構文のみを使用してください。**

    *   HTMLタグは使用しないでください。

## Markdown構造ルール

*   各スライドは `# 見出し` で始めてください。
*   `##` 以下のレベルの見出しはスライドの区切りにならないため、使用しないでください（またはスライド内の要素として扱われます）。
*   スライド本文は以下のみを使用して構成してください：
    *   見出し
    *   箇条書きリスト
*   **スライド本文の終わりと `::: notes` の間には、必ず1行以上の空行を入れてください。**
    *   箇条書き、画像、段落の直後に `::: notes` を書かないでください。空行がない場合、notes ブロックがスライド本文の一部として表示されてしまいます。

*   読み上げ原稿は **厳密に以下の形式で** 書いてください：

::: notes
(このスライドで読み上げるテキストを、スピーチに適した形式で記述してください。)
:::

*   **スライドに表示されていない補足情報や話し言葉** はすべて notes に含めてください。
*   スライド本文と notes の内容は **重複しても構いません**（スライドは要約、notesは詳細、という形が望ましいです）。

## 出力フォーマット

*   **Markdown のみを出力してください。**
*   説明、前置き、コメントは書かないでください。
*   ファイル全体を単一の Markdown として出力してください。

## ドキュメント
(ここに動画化したいドキュメントやテキストを貼り付けてください)
```

## 生成された出力の使い方

1.  AI が生成した Markdown コードブロックをコピーします。
2.  テキストファイル（例: `ai_project.md`）に貼り付けて保存します。
3.  `slidemovie ai_project -p` を実行してスライドを生成します。
4.  あとは標準的なワークフローに従って、デザイン編集と動画生成を行ってください。

## 生成結果の例

実際に上記のプロンプトを使用して AI に生成させた Markdown ファイルと、それをもとに作成された動画の例です。

**完成動画:**
[https://www.youtube.com/watch?v=mlj-_Tyhn0Q](https://www.youtube.com/watch?v=mlj-_Tyhn0Q)

**生成された Markdown ソース:**

```markdown
# slidemovieの紹介

::: notes
slidemovieの紹介へようこそ。本日は、このプログラムの目的と、プレゼンテーション動画の作成方法をどのように革新できるかについて説明します。
:::

# 動画作成の課題

-   ナレーションの録音が難しい
-   動画編集に時間がかかる
-   音声とスライドの同期が面倒
-   内容の更新には再録音が必要

::: notes
プロフェッショナルなプレゼンテーション動画を作成するのは、しばしば困難を伴います。通常、自分の声を録音する必要があり、これには静かな環境が必要で、緊張することもあります。さらに、スピーチとスライドを手動で同期させるために何時間もの動画編集が必要です。その上、後で一文だけ変更したい場合でも、多くの場合、プロセス全体を最初からやり直さなければなりません。
:::

# slidemovieとは？

-   動画制作を自動化
-   Markdown、PowerPoint、AIを結合
-   動画編集ソフト不要
-   録音機材不要

::: notes
slidemovieはこれらの問題に対する解決策です。これは、Markdownテキストの構造的な単純さ、Microsoft PowerPointのデザイン機能、そして人工知能の力を組み合わせることで、動画制作を自動化する強力なツールです。このプログラムを使えば、複雑な動画編集ソフトや高価な録音機材は必要ありません。
:::

# ステップ1: 原稿を書く

-   シンプルなテキストファイル（Markdown）を使用
-   スライドの内容を定義
-   ナレーション原稿を記述

::: notes
ワークフローは非常にシンプルです。まず、Markdownを使用して標準的なテキストファイルにコンテンツを記述します。この単一のファイル内で、スライドに表示される内容を正確に定義し、ナレーターが読み上げる原稿を記述します。
:::

# ステップ2: スライドをデザインする

-   PowerPointファイルを自動生成
-   PowerPointの「デザイナー」機能を使用
-   プロ仕様のレイアウトを瞬時に適用

::: notes
次に、プログラムがテキストをドラフトのPowerPointファイルに変換します。その後、このファイルを開き、PowerPointに組み込まれたAIデザイナーを使用して、プロ仕様のレイアウトやビジュアルを瞬時に適用できます。これにより、ゼロからデザインすることなく、プレゼンテーションの外観を完全にコントロールできます。
:::

# ステップ3: 動画を生成する

-   コマンドを1つ実行
-   AIが自然なナレーション音声を生成
-   スライドを画像に変換
-   音声と映像を自動同期

::: notes
最後に、コマンドを1つ実行して動画を作成します。プログラムは高度なAIを使用して、原稿から自然な響きのナレーション音声を生成します。そして、音声とスライド画像を自動的に結合し、タイミング調整や同期をすべて処理します。
:::

# 主な利点

-   動画をコードのように扱える
-   バージョン管理が容易
-   差分更新が可能
-   時間と労力を節約

::: notes
最大の利点の1つは、動画をコードのように扱えることです。スライドを更新する必要がある場合は、テキストファイルを編集して再ビルドするだけです。プログラムは変更された部分のみを再生成するほどスマートなので、従来の動画編集に比べて大幅な時間を節約できます。
:::
```
